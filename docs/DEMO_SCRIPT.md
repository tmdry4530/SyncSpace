# SyncSpace 데모 시연 대본

이 문서는 발표 중 실제 앱을 조작하면서 말할 짧은 시연 대본이다. 전체 설명보다 “사용자가 어떤 흐름으로 쓰는지”를 보여주는 데 집중한다.

## 데모 목표

1. 워크스페이스 안에서 채팅과 문서가 한 화면에 있는지 보여준다.
2. 채널/문서 전환이 workbench 맥락을 유지하는지 보여준다.
3. 문서 작성에서 `/`, `[[문서명]]`, `#태그` 흐름을 보여준다.
4. 사이드바 접힘/반응형 UI가 작업 흐름을 방해하지 않는지 보여준다.

## 0. 준비

- 앱 접속: `https://sync-space-green.vercel.app/`
- 로그인 후 워크스페이스가 하나 이상 있는 상태로 시작한다.
- 데모용 채널과 문서가 없다면 미리 하나씩 만들어둔다.

## 1. 워크스페이스 진입

**화면 조작**

워크스페이스 목록에서 데모 워크스페이스로 들어간다.

**멘트**

“워크스페이스에 들어오면 왼쪽에는 채널과 문서 목록이 있고, 오른쪽에는 채팅과 문서가 동시에 보이는 workbench가 나타납니다. 이 화면이 SyncSpace의 핵심 화면입니다.”

## 2. 채팅과 문서가 같은 화면에 있는 점 강조

**화면 조작**

채팅 입력창에 짧은 메시지를 입력한다.

예시:

```txt
오늘 회의 결정사항을 오른쪽 문서에 정리하겠습니다.
```

**멘트**

“왼쪽에서는 채널 대화를 이어가고, 오른쪽에서는 바로 문서를 작성할 수 있습니다. 대화와 문서가 분리된 화면이 아니라 같은 작업 맥락 안에 있습니다.”

## 3. 문서 작성 흐름 시연

**화면 조작**

문서 영역에 내용을 입력한다.

예시:

```txt
# 회의 결정사항

- 이번 주 목표 정리
- 담당자 배정

#todo 다음 회의 전까지 초안 작성
```

필요하면 `/`를 입력해 슬래시 명령 메뉴가 뜨는 것을 보여준다.

**멘트**

“문서 편집은 별도 toolbar를 많이 노출하기보다 Markdown과 slash command 중심으로 만들었습니다. 그래서 작성 흐름이 끊기지 않습니다.”

## 4. 문서 링크 시연

**화면 조작**

문서에 아래 형태를 입력한다.

```txt
[[기획 메모]]
```

오른쪽 knowledge rail의 문서 링크 후보 또는 매칭 결과를 보여준다. 이미 존재하는 문서 링크가 있다면 클릭한다.

**멘트**

“문서 안에서 `[[문서명]]`을 쓰면 같은 워크스페이스의 문서 링크로 인식합니다. 링크를 눌러도 전체 페이지가 새로고침되는 것이 아니라, React Router 안에서 workbench 맥락을 유지한 채 문서만 전환됩니다.”

## 5. 채널/문서 전환 시연

**화면 조작**

사이드바에서 다른 채널 또는 다른 문서를 클릭한다.

**멘트**

“채널과 문서는 URL에도 반영됩니다. 그래서 새로고침하거나 링크를 공유해도 현재 보고 있던 작업 조합을 복원할 수 있습니다.”

## 6. 사이드바 접힘 시연

**화면 조작**

사이드바 접기 버튼을 누른다. 접힌 상태에서 채널/문서 구분과 생성 버튼을 보여준다.

**멘트**

“사이드바를 접어도 채널과 문서 구분이 사라지지 않게 했습니다. 작업 공간을 넓히면서도 이동과 생성 동작은 유지됩니다.”

## 7. 실시간 상태 요약 언급

**화면 조작**

상단 command bar의 `실시간 연결 중`과 접속 인원 표시를 가리킨다.

**멘트**

“상단 상태는 개별 채팅 room이나 문서 room 하나의 상태가 아니라, 워크스페이스에서 실시간 작업이 가능한지를 보여주는 요약입니다.”

## 8. 마무리

**멘트**

“정리하면, SyncSpace는 채팅과 문서 작성을 한 화면에 묶어 협업 흐름을 줄이는 프로젝트입니다. 구현 측면에서는 Zustand, TanStack Query, Yjs로 상태 책임을 분리했고, React Router로 작업 맥락을 URL에 유지했습니다.”
