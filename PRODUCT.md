# SyncSpace Product Context

## Register

Product UI first, portfolio storytelling second.

## Product purpose

SyncSpace is a real-time collaborative workbench where a team can discuss decisions in chat and write the shared document in the same screen. The portfolio value is the clear separation of local UI state, server state, and realtime collaboration state.

## Users

- A reviewer evaluating a React portfolio project.
- A small product or study team coordinating decisions across chat and documents.
- The project owner who wants the frontend structure to demonstrate practical state-management choices.

## Core promise

A workspace should feel like a focused cockpit: channels, documents, presence, invite code, and account controls are available without covering the work.

## Design principles

- Keep the workbench calm and dense. Controls should support the task, not compete with content.
- Make sparse states intentional. Early accounts with one or zero workspaces should still feel guided and complete.
- Keep navigation predictable. Workspace switching should be a visible top-level action, while account menus hold account-only actions.
- Prefer quiet product polish over decorative effects.

## Anti-references

- Generic SaaS dashboards with oversized empty grids.
- Loud floating controls that draw attention away from chat and document work.
- Hidden navigation trapped inside account menus.
- Decorative scrollbars in message areas.
