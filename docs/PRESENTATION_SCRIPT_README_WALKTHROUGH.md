# SyncSpace 발표 대본

이 대본은 README를 화면에 띄워두고 짧게 설명하는 흐름이다. 목표는 모든 구현을 설명하는 것이 아니라, **왜 만들었는지**, **핵심 UI가 무엇인지**, **상태를 어떻게 나눴는지**만 분명히 전달하는 것이다.

## 1. 시작

안녕하세요. 제가 발표할 프로젝트는 **SyncSpace**입니다.

SyncSpace는 채팅에서 결정한 내용을 같은 워크스페이스의 문서에 바로 정리할 수 있게 만든 실시간 협업 워크벤치입니다.

일반적인 협업 도구에서는 대화와 문서가 분리되어 있어서, 채팅에서 나온 결정을 다시 문서로 옮기는 과정에서 맥락이 끊길 수 있습니다. 이 프로젝트는 그 흐름을 하나의 화면에 묶는 것을 목표로 했습니다.

## 2. 핵심 화면

가장 중요한 화면은 **split workbench**입니다.

왼쪽에는 채널 채팅이 있고, 오른쪽에는 문서 편집기가 있습니다. 사용자는 채팅을 보면서 바로 문서를 작성할 수 있습니다.

상단 command bar에는 현재 채널과 문서, 실시간 연결 상태, 접속 인원이 모여 있습니다. 반대로 각 패널 안에서는 `채팅`, `문서`라는 역할만 보여줘서 화면 정보가 중복되지 않게 했습니다.

사이드바에서는 워크스페이스 안의 채널과 문서를 전환할 수 있고, 접은 상태나 모바일 상태에서도 같은 작업 흐름이 유지되도록 했습니다.

## 3. 상태 관리 설계

이 프로젝트에서 가장 신경 쓴 부분은 상태를 나누는 것입니다.

먼저 Zustand는 UI 상태를 담당합니다. 예를 들면 사이드바가 접혔는지, 현재 어떤 채널과 문서를 보고 있는지, 채팅 입력 draft가 무엇인지 같은 값입니다.

TanStack Query는 서버 상태를 담당합니다. 워크스페이스, 채널, 문서, 메시지 히스토리처럼 서버에서 가져오고 캐싱해야 하는 데이터입니다.

Yjs는 실시간 협업 상태를 담당합니다. 채팅 room, 문서 room, presence처럼 여러 브라우저 사이에서 동기화되어야 하는 상태입니다.

이렇게 나누면 React 컴포넌트가 UI 상태, 서버 캐시, WebSocket 동기화를 한꺼번에 떠안지 않아도 됩니다.

## 4. 문서 편집 흐름

문서 편집은 toolbar보다 Markdown과 shortcut 중심으로 만들었습니다.

`/` 명령으로 제목이나 목록 같은 블록을 넣을 수 있고, `[[문서명]]`을 쓰면 같은 워크스페이스의 문서 링크로 연결됩니다. `#태그`도 문서의 맥락 정보로 수집됩니다.

문서 링크는 React Router의 `Link`를 사용합니다. 그래서 문서를 이동해도 전체 페이지가 새로고침되지 않고, 현재 workbench 안에서 문서만 전환됩니다.

## 5. 실시간 구조

실시간 room은 채팅과 문서를 분리했습니다.

```txt
chat:{workspaceId}:{channelId}
doc:{workspaceId}:{documentId}
```

채팅은 메시지 중심이고, 문서는 editor document 중심이라 데이터 성격이 다릅니다. 그래서 room을 분리해 lifecycle을 독립적으로 관리했습니다.

사용자에게 보이는 상단 상태는 개별 room 상태를 모두 엄격히 합친 값이 아니라, 워크스페이스에 접속해 실시간 작업을 할 수 있는지를 보여주는 요약입니다.

## 6. 기술 스택

프론트엔드는 React 19, TypeScript, Vite를 사용했습니다.

라우팅은 React Router, 로컬 상태는 Zustand, 서버 상태는 TanStack Query, 실시간 협업은 Yjs와 y-websocket을 사용했습니다.

문서 편집기는 Tiptap 기반이고, backend는 Node.js HTTP/WebSocket 서버입니다. 인증과 데이터베이스는 Supabase Auth, Postgres, RLS를 사용했습니다.

## 7. 마무리

정리하면 SyncSpace는 채팅과 문서화를 한 화면에 합친 React 협업 앱입니다.

제가 보여주고 싶은 핵심은 두 가지입니다.

첫 번째는 사용자 흐름입니다. 대화와 문서 작성을 분리하지 않고 하나의 workbench에서 처리했습니다.

두 번째는 구조입니다. UI 상태, 서버 상태, 실시간 협업 상태를 각각 Zustand, TanStack Query, Yjs로 분리했습니다.

이 프로젝트를 통해 단순한 CRUD 화면이 아니라, 실제 협업 앱에서 필요한 라우팅, 상태 관리, 실시간 동기화, editor integration을 함께 구현해봤습니다.
